## 1. System Overview

**Actinova AI Tutor** is a SaaS learning platform that uses OpenAI to:
- 📚 Generate structured courses from any topic
- 🤖 Provide an interactive AI tutor chat
- 📝 Create academic reports in MLA, APA 7, or Chicago citation styles
- 🧪 Generate and track quizzes

Users authenticate via JWT cookies, are gated by a subscription plan (Free / Pro / Enterprise), and all AI-generated content is stored in MongoDB.

## 3. Authentication

Authentication uses **JWT tokens** stored in **HttpOnly cookies**.

### Flow
1. User signs up or logs in at `/api/auth/login` or `/api/auth/register`
2. A signed JWT (`JWT_SECRET`) containing `{ userId, email }` is set as a cookie named `token`
3. Every protected API route passes through `withAuth` middleware which verifies the JWT and attaches `request.user`

### Middleware Chain

```js
// Example route
export const POST = combineMiddleware(
  withErrorHandling,  // wraps in try/catch, logs errors
  withCsrf,           // validates CSRF token from header + cookie
  withAuth,           // verifies JWT, attaches request.user
  (h) => withAPIRateLimit(h, "feature-name")  // rate limits per feature
)(handlePost);
```

### CSRF Protection
- Tokens generated by `/api/csrf` and stored in a cookie + sent as a request header `x-csrf-token`
- `withCsrf.js` uses `await import("next/headers")` dynamically to prevent Next.js build-time errors

## 5. Feature Modules

### 5.1 Course Generation

**Route:** `POST /api/generate-course`  
**File:** `src/app/api/generate-course/route.js`

Generates a structured multi-module course from a topic. Supports quizzes, flashcards, and lesson content through a single prompt.

**Request body:**
```json
{
  "topic": "Machine Learning",
  "difficulty": "intermediate",
  "modules": 3,
  "lessonsPerModule": 3
}
```

**Checks:** Plan limits (`generateCourseLimit`), rate limiting, auth.

---

### 5.2 AI Tutor Chat

**Route:** `POST /api/chat`  
**File:** `src/app/api/chat/route.js`

Provides an interactive AI tutor strictly focused on a topic. Rejects off-topic questions.

**Request body:**
```json
{
  "message": "Explain gradient descent",
  "topic": "Machine Learning",
  "conversationHistory": [...]
}
```

**Limits:** 3 AI responses/day on Free tier.

**Routes:** `GET/POST /api/quizzes`  
**Model:** `src/app/models/Quiz.js`

Generates multiple-choice / short-answer quizzes based on a topic and difficulty. Results stored per user.

## 6. Report Editor — Deep Dive

The Report Editor is the most complex feature. It enables users to generate full academic papers section-by-section with a rich in-browser editor.

### 6.1 Report Lifecycle

```
1. User fills out a report form (topic, type, citation style, length)
        ↓
2. POST /api/generate-report-outline
   → GPT generates a structured outline (sections with titles/descriptions)
   → Outline saved to MongoDB reports collection
        ↓
3. User is redirected to /reports/[id] (ReportEditor component)
        ↓
4. User clicks "Draft" on each section card in the Outline panel
   → POST /api/generate-report-section for each section
   → Content appended to the editor
   → References aggregated into the References page
        ↓
5. Everything autosaves to DB every 2 seconds (debounced)
```

### 6.2 Citation Styles

| Style | Title Page Format |
|-------|-------------------|
| APA 7 | Centered: title, author name, institution, course, date |
| MLA | Top-left block: name, instructor, course, date; centered title |
| Chicago | Centered: title, author, institution, course, date |

The title page is generated as a non-editable `div#title-page` at the top of the editor content.

### 6.3 Persistence (Autosave)

Every 2 seconds after any edit, the following fields are saved to MongoDB via `PATCH /api/reports/[id]`:

| Field | Description |
|-------|-------------|
| `fullContent` | Full editor innerHTML (all pages) |
| `sections` | Map of `{ sectionId: true }` — tracks which sections are drafted |
| `references` | Array of formatted reference strings |
| `abstract` | Plain text abstract |
| `title` | Report title |
| `outline` | Section structure array |
| `sectionLengths` | Per-section page count preferences |
| `authorName` | Author / student name |
| `institution` | Institution name |
| `courseName` | Course name |
| `studentName` | Student name |
| `submissionDate` | Formatted submission date |

On load, **all fields** are restored from DB so users resume exactly where they left off — including which sections are marked as "done" in the outline cards.

**Autosave guard:** A `loadedRef` ref prevents the autosave debounce from running during the initial content restore, avoiding unnecessary saves or race conditions.

**Autosave indicator:** The toolbar shows:
- 🔄 `Saving…` with a spinner while the request is in flight
- ✅ `Saved HH:MM` after each successful save
- `Ctrl/Cmd+S` triggers an immediate manual save

### 6.4 Toolbar Features

| Control | Function |
|---------|---------|
| Font family selector | Applies font family to selected text |
| Font size stepper (−/number/+) | Any size 8–72px applied via CSS span |
| Line height selector | 1.0, 1.15, 1.5, 2.0, 2.5, 3.0 |
| Bold / Italic / Underline | execCommand formatting |
| Align Left / Center / Right / Justify | execCommand alignment |
| Bullet / Numbered list | execCommand list |
| Zoom −/+ | CSS `transform: scale(zoom%)` on document |
| Outline toggle (⊞) | Shows/hides outline panel |
| Save (💾) | Immediate manual save |
| Download (⬇) | Exports `.docx` file via server |

### 6.5 Responsive Design

- **Mobile (< xl):** Outline hidden; toggle button opens a bottom-drawer slide-up panel
- **Desktop (xl+):** Floating outline sidebar on the right, connected by dashed SVG lines
- Toolbar scrolls horizontally on small screens
- Document page uses `w-full max-w-[260mm]` with responsive padding

### 6.6 DOCX Export

**Route:** `POST /api/export-docx`  
Converts the report's content to a `.docx` file using the `docx` npm library on the server. The file is streamed back and saved with `file-saver` on the client.

## 8. Database Schema

### `reports` Collection

```js
{
  _id: ObjectId,
  userId: ObjectId,          // ref: users
  title: String,             // report title
  topic: String,             // original topic
  type: String,              // "Research Paper", "Essay", etc.
  citationStyle: String,     // "APA 7" | "MLA" | "Chicago"
  academicLevel: String,     // "Undergraduate" | "Graduate" | etc.
  criticalDepth: String,     // "Moderate" | "Deep" | etc.
  outline: Array,            // [{ id, title, description }]
  sections: Object,          // { sectionId: true } — completion map
  fullContent: String,       // full editor innerHTML
  abstract: String,          // plain text abstract
  references: Array,         // formatted reference strings
  sectionLengths: Object,    // { sectionId: 1|2|3 } — target pages
  // metadata
  authorName: String,
  institution: String,
  courseName: String,
  studentName: String,
  submissionDate: String,
  createdAt: Date,
  updatedAt: Date,
}
```

### `users` Collection

```js
{
  _id: ObjectId,
  email: String,
  passwordHash: String,
  firstName: String,
```

## 9. Middleware & Security

### Middleware Stack (`src/app/lib/middleware.js`)

| Middleware | Function |
|-----------|----------|
| `withErrorHandling` | Wraps handler in try/catch, returns JSON error |
| `withAuth` | Verifies JWT, blocks unauthenticated requests |
| `withCsrf` | Validates CSRF token for mutating requests |
| `withAPIRateLimit` | Feature-level rate limiting stored in MongoDB |
| `combineMiddleware` | Composes middleware functions right-to-left |

### CSRF

- Token generated at `/api/csrf` (GET), stored as cookie `csrf-token`
- Every mutating request must include `x-csrf-token` header matching the cookie
- `withCsrf.js` uses `await import("next/headers")` to avoid Next.js build-time evaluation errors

### Rate Limiting

Tracked in MongoDB `users.apiUsage` per feature per month. Enforced in `src/app/lib/planMiddleware.js`.

## 11. Build & Deployment

### Development

```bash
npm install
npm run dev        # Starts Next.js dev server on :3000
```

### Production Build

```bash
npm run build      # Compiles and optimizes
npm start          # Starts production server
```

> **Note:** API routes that call `next/headers` must either:
> - Export `export const dynamic = "force-dynamic";` at the top, OR  
> - Use `await import("next/headers")` inside the handler (not at module scope)  
>
> This prevents Next.js from attempting to statically pre-render routes that need request context.

### Key Build Fixes Applied

| File | Fix |
|------|-----|
| `src/app/lib/withCsrf.js` | Moved `next/headers` import inside handler using dynamic `await import()` |
| `src/app/lib/middleware.js` | Wrapped `nextHeaders()`/`nextCookies()` calls in try/catch |
| `src/app/api/generate-course/route.js` | Added `export const dynamic = "force-dynamic"` |
| `src/app/api/session/start/route.js` | Added `export const dynamic = "force-dynamic"` |
| `src/app/api/billing/webhook/route.js` | Added `export const dynamic = "force-dynamic"` |
