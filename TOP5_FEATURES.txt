Top 5 Recommended Features for Actirova AI Tutor
Generated: 2026-07-13T20:21:23+03:00

1. In-App Notification Center + Scheduled Push Notifications
   Impact: High | Effort: Medium
   The platform has Web Push and browser Notification APIs, plus a UserActivity audit log, but no centralized notification feed. Users have no way to see "Your course was shared" or "Streak warning" inside the app. Implement a Notification model in MongoDB with read/unread state, a notification bell in the sidebar, and a cron job that sends scheduled study reminders, streak warnings, and daily login bonuses via the existing /api/push/send endpoint. This directly improves retention and daily active usage.

2. Time-Range Analytics + Dashboard Export
   Impact: High | Effort: Low
   All analytics APIs (/api/analytics/overview, activity-heatmap, study-time, quiz-trends) return all-time data with no startDate/endDate filters. Users cannot see "my progress this month" or export a progress summary. Add query params to the analytics routes, update DashboardOverview to include a date range picker, and add PNG/PDF export for the activity heatmap and mastery summary. Low effort because the aggregation logic already exists; only filtering and export wrappers are needed.

3. PDF Export for Reports + Citation Manager
   Impact: High | Effort: Low-Medium
   ReportEditor currently exports only DOCX via /api/generate-doc. Many users need PDF for academic submission. Add PDF export using the already-installed @react-pdf/renderer or jspdf. In parallel, replace the basic references panel with a citation manager that supports APA/MLA/Chicago formats, DOI lookup, and in-text citation insertion. This closes a critical gap for the academic writing workflow.

4. Unified Search API with Full-Text Support
   Impact: High | Effort: Medium
   Search is currently O(n) client-side filtering or simple includes() on the server. There is no /api/search endpoint, no MongoDB text indexes, no relevance ranking, and no search across the full content library (courses, study plans, flashcards, quizzes, blog posts, reports). Add a dedicated /api/search route with MongoDB Atlas Search or Meilisearch integration, faceted filters (difficulty, category, date), and autocomplete suggestions. This is the backbone for improved content discovery everywhere.

5. Streak Freeze + Daily Login Bonus + Badges Page
   Impact: High | Effort: Low
   The gamification system has XP rewards, 21 badge definitions, and a level curve, but daily_login (5 XP) is never awarded, there is no streak recovery mechanism, and GamificationBadge.jsx exists but is not wired into any visible page. Users cannot browse their badge collection. Implement daily login detection with escalating bonuses, a consumable streak freeze (earned via milestones or credits), and a dedicated /profile/badges page. This unlocks the existing gamification infrastructure with minimal new code.
